Let's get into If Statements. An if statement (sometimes called a conditional or a logical operator) can be the most crucial part of any code. It can check one or more parameters for information, and depending on the information that those parameters contain, the if statement can direct all sorts of things to happen. Imagine if you wanted something to happen in your program but only if other parameters are met. This is where an if statement comes into play.

One of the key elements of an if statement is that an if statement will always evaluate to a boolean result (true/false), which you will find extremely useful in writing code.

Let's think of an example. Imagine you want to add two numbers togther, but only if one of the numbers is 5. So if you have input of 7 and 19, the function would not run. If the input is 6 and 5, then the function would run. This is a fairly basic (and slightly useless) example, but it gets the point across.

If statements are constructed as follows:

if (some parameter is true) {

   run this code;

   }

 

So, in our example above, the code would translate to something like this...

if (one of the inputs is 5) {

   add the inputs;

   }

 

Let's watch an example in action. First, we will introduce a variable. The single = is for setting a value, while the triple === is for checking equality. You can see that we are first setting x to be 10, and then we are checking if x is equal to 10. Hit the green play button to watch it run!

Let's throw some variables into the mix. Variables are where you can store data for later use. You can also change variables. Let's say you had a variable with a value of 50 (var x = 50). Now, let's say you used that variable throughout your code in potentially hundreds or even thousands of places! If you did not have a variable, you'd have to type '50' each time. Not the biggest deal, it's just one more character than it would be to type 'x', BUT what happens if you suddenly need to change that value to be 40 everywhere it's used? If you did not use a variable, you'd have to update in hundreds, if not thousands, of places with the possibility of missing some. If you did use a variable, you just need to change it in one place!
